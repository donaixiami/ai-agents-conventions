# 通用 UI 出稿验收清单

适用于本仓库**任意** UI / 原型 / 视觉稿交付。与具体业务页无关。

## 输入

- [ ] 当轮目标、范围、端、验收标准已明确
- [ ] 相关产品/UI 规范或最小 token 表已读（或已新建最小约定）

## 结构

- [ ] 可本地打开预览，无「必须后端才出静态结构」
- [ ] 无多层 iframe + 实体转义套娃
- [ ] 用户可见界面无作者/评审/实现过程元叙事

## 视觉与系统

- [ ] 使用统一 token（色/字号/间距/圆角），无满页魔法数
- [ ] 状态有文字或明确语义，非仅颜色
- [ ] 图标体系统一（SVG 或项目约定），无 emoji 充业务图标

## 内容与数据

- [ ] 演示数据全链路自洽
- [ ] 金额/数量等格式统一
- [ ] 文案面向最终用户

## 交互

- [ ] 任务所需关键态可演示（含弹层/抽屉若在需求内）
- [ ] 危险/写操作用显式动作标识确认，非文案 includes 猜测
- [ ] 多端时业务一致，仅壳差异

## 可审查（若用户要求量测/切元素）

- [ ] 真实可点选元素（DOM 或 Figma 图层）
- [ ] 能查看宽高、边距、颜色、字体等
- [ ] 未用整页 PNG 冒充可审查源

## 导出 / Figma（若需要）

- [ ] 矢量图标与位图职责分离
- [ ] Figma 复用已有文件（用户要求时）；限流有降级说明

## 不规则运行时切图（可选 · 必须先问）

- [ ] **已询问用户**是否导出（未问禁止导出）
- [ ] 用户同意或明确范围后才执行；拒绝/未表态则跳过
- [ ] 仅导出不规则/跨端易漂视觉，未把普通卡片列表切成大图
- [ ] 已给出 `static` / `ui-assets` 路径与引用示例，并标明仍应用 CSS 的部分

## 交付

- [ ] 中文说明：产物路径、如何打开、如何验证
- [ ] 未完成项与阻塞写清
