# 多项目上下文盘点清单

## 触发

- [ ] 已判定管辖范围内独立项目数（是否 ≥ 3）
- [ ] ≥3 时走完整盘点；&lt;3 时至少核对当次相关项目

## 公共层

- [ ] 存在 `docs/context/REGISTRY.md`（或等价登记表）
- [ ] 登记表覆盖全部独立项目（无遗漏目录）
- [ ] 每项有：路径、CONTEXT 状态、skill 状态、核对日期

## 项目层（每个项目）

- [ ] `docs/PROJECT_CONTEXT.md` 存在且非空壳（定位 + 栈/目录 + 验证 至少两类）
- [ ] 项目 skill 存在且含 First Steps / 必读文档
- [ ] skill 指向该项目的 PROJECT_CONTEXT
- [ ] 架构性变更后日期或内容已更新（非明显过期）

## 缺失处理

- [ ] 已深度阅读代码与现有文档后再写
- [ ] 不确定点标「待确认」，未编造契约
- [ ] 新建/补齐后已回写 REGISTRY

## 恢复会话

- [ ] 先 REGISTRY，再相关项目 CONTEXT + skill
- [ ] 未跳过盘点直接跨项目改代码
